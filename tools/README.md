# 🛠️ Python SDK Maintenance Guide
##
**Title:**  
Robust Release Workflow for `ioi-swarm`

**Summary:**  
To prevent version mismatches between Git tags and PyPI distributions, this guide enforces a "Commit-Then-Tag" sequence. This ensures that the build artifacts generated by GitHub Actions always contain the correct version metadata.

**Workflow Components:**  
*   `ioi-swarm/python/pyproject.toml` (Source of Truth)
*   `.github/workflows/workflow.yml` (CI/CD)

---

## 🚀 Standard Release Procedure

Follow these steps exactly to publish a new version.

### 1. Update Metadata
Edit `ioi-swarm/python/pyproject.toml` and increment the version number.
```toml
version = "0.1.x"
```

### 2. Local Verification
Verify that the build system sees the correct version before pushing.
```bash
cd ioi-swarm/python
hatch build
# Check the filename in dist/
ls dist/
```

### 3. Commit to Master
**CRITICAL:** You must push the file change to the remote repository *before* creating the tag.
```bash
git add pyproject.toml
git commit -m "release: version 0.1.x"
git push origin master
```

### 4. Tag and Trigger
Create the version tag on the commit that contains the updated `pyproject.toml`.
```bash
git tag v0.1.x
git push origin v0.1.x
```

---

## 🆘 Troubleshooting: Fixing a Botched Release

If a GitHub Action succeeds but the version is not appearing on PyPI (usually due to a tag pointing to an old version of `pyproject.toml`), use the **Force Sync** method:

### 1. Correct the file
Ensure `pyproject.toml` has the desired version and is committed/pushed to `master`.

### 2. Purge the stale tag
```bash
# Delete locally
git tag -d v0.1.x

# Delete on GitHub
git push origin :refs/tags/v0.1.x
```

### 3. Re-Tag and Re-Push
```bash
git tag v0.1.x
git push origin v0.1.x
```

### The Concept: "Tool-to-Artifact" Pipeline

1.  **Chat Stream (The "Log"):** Displays the *intent* ("I will write the file") and the *status* ("File saved").
2.  **MCP/Tool Layer (The "Action"):** The LLM calls `fs.write_file` or `create_component`.
3.  **Artifact Panel (The "Result"):** Instead of dumping 500 lines of code into the chat bubble, the UI intercepts that specific tool payload and renders it in the side panel.

  // ... existing state
  const [currentArtifact, setCurrentArtifact] = useState<{name: string, content: string, language: string} | null>(null);

  // ... existing hooks

  // ------------------------------------------------------------
  // THE NEW SCAFFOLDING LOGIC
  // ------------------------------------------------------------
  useEffect(() => {
    if (!task) return;

    // 1. Get the latest message
    const lastMsg = task.history[task.history.length - 1];

    // 2. Check if it is a Tool Call (Agent -> System)
    // In a real scenario, you might parse the raw 'tool_calls' array from the LLM response
    // For now, we simulate detecting a specific intent or tool usage
    
    const isCodeGeneration = 
      lastMsg?.role === "agent" && 
      (lastMsg?.text.includes("```") || task.current_step.includes("write_file"));

    if (isCodeGeneration) {
      // 3. Extract content (Regex to grab code blocks or file paths)
      // This is a simple parser, but you could parse the actual MCP JSON payload
      const codeBlockMatch = lastMsg.text.match(/```(\w+)?\n([\s\S]*?)```/);
      
      if (codeBlockMatch) {
        const language = codeBlockMatch[1] || "typescript";
        const content = codeBlockMatch[2];
        
        // Check if artifact is new or different
        if (content !== currentArtifact?.content) {
            setCurrentArtifact({
                name: "generated_output." + (language === "python" ? "py" : "ts"), // You'd parse the real filename from the tool args
                language: language,
                content: content
            });
            setShowArtifactPanel(true);
            
            // Optional: Auto-expand window if it's too narrow
            if (containerWidth < 800) {
               invoke("resize_spotlight", { width: 900, height: window.innerHeight }).catch(console.error);
               setContainerWidth(900);
            }
        }
      }
    }
  }, [task?.history, task?.current_step]);
