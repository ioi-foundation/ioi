# CIRC/CEC v1 Collapse Semantics Alignment

Status: alignment report.
Canonical owner: this file for the 2026-05-04 CIRC/CEC v1 documentation pass.
Supersedes: none.
Superseded by: future conformance alignment reports.
Last alignment pass: 2026-05-04.

## Executive Verdict

CIRC and CEC already contained the core runtime invariants for converting
probabilistic model inference into deterministic runtime effects. This pass
promotes that latent structure into explicit v1 language without introducing a
new product-facing concept or mandatory runtime object.

## What Changed

- CIRC is now framed as the semantic collapse contract: uncertain user intent
  resolves to a deterministic selected intent, abstention, or resolver failure.
- CEC is now framed as the effect-collapse contract: bounded pre-effect
  synthesis resolves to one approved, single-shot execution attempt and a
  receipt-backed terminal state.
- The docs make explicit that failures after collapse must not reopen semantic
  ranking or heuristic execution retries.
- Implementation carriers such as route frames, decision records, context
  envelopes, plan refs, and receipt bundles remain implementation details. The
  invariant is the collapse discipline itself.

## Vocabulary Boundary

`CIRC`, `CEC`, and `canonical collapse` remain conformance/spec vocabulary.
Runtime code and product UI should continue to use behavior-first names such as
intent, policy, decision record, plan, approval, receipt, verification, and
completion gate.

## Validation

This was a documentation-only alignment pass. No runtime behavior changed.
