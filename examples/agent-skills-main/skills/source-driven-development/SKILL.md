# Source Driven Development

Use this skill when a workflow must be anchored in source material rather than memory or assumptions.

1. Prefer primary repo files, official docs, schemas, tests, and generated contracts.
2. Use fast search to find the smallest relevant source set.
3. Cite the files or artifacts that drove the decision.
4. Treat unstable external facts as requiring fresh verification.
5. Keep conclusions tied to evidence the workflow can replay.
