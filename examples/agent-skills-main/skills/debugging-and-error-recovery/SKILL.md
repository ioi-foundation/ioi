# Debugging And Error Recovery

Use this skill when a workflow must turn a failing or flaky behavior into a diagnosed, verified fix.

1. Capture the failing command, input, or trace.
2. Reproduce the failure or explain clearly why it cannot be reproduced.
3. Localize the smallest responsible boundary.
4. Patch the cause without hiding diagnostics.
5. Verify both the fixed path and the relevant regression guard.
