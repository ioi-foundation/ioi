# Test Driven Development

Use this skill when a workflow needs regression coverage tied directly to a requested behavior.

1. State the expected behavior in executable form.
2. Add or update the smallest focused test that proves the behavior.
3. Run the test and confirm the initial failure when practical.
4. Implement the smallest production change needed.
5. Rerun the targeted test and record the result.
