# Code Review And Quality

Use this skill when a workflow needs review-quality judgment rather than implementation first.

1. Lead with findings ordered by severity.
2. Ground each finding in a file path and concrete behavior.
3. Prefer bugs, regressions, data loss, security, and missing test risks.
4. Keep summaries secondary to actionable findings.
5. State residual risk or test gaps when no issue is found.
