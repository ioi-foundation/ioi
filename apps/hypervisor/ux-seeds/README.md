# Hypervisor UX Seeds

This directory preserves the source-neutral interaction and layout work from
PRs #93–#95 without activating later product stages.

These files are dormant evidence. They are deliberately absent from the
surface registry, application catalog, route dispatcher, active parity matrix,
and active pixel-certification census. The retained pixel files prove only the
historical shell comparison at their recorded viewports. They do not prove
current owner truth, product membership, authority, operational depth, or an
M-stage exit.

Promotion is a new owner-led implementation decision. It must begin from the
current M-stage contract and product taxonomy, revalidate every daemon binding,
declare honest capabilities, and pass the current focused and adversarial bars.
The seed must never be promoted merely because its historical screenshot or
pixel comparison passed.

Run the anti-promotion guard with:

```sh
node apps/hypervisor/scripts/verify-hypervisor-ux-seed-evidence.mjs
```
